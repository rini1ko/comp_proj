# Tree-Based Database Engine

Навчальний проєкт: маленька база даних, яка зберігає записи в одному з шести різних дерев. Можна обрати дерево і подивитись, як воно змінюється після кожної операції (зберігається GIF). А ще є окремий скрипт, який порівнює, яке дерево швидше.

## Що ми реалізували

- 6 дерев: AVL, Splay, червоно-чорне (RB), 2-3, B-дерево, B+ дерево
- Просту базу даних з командами `INSERT`, `SELECT`, `UPDATE`, `DELETE`
- Візуалізацію дерева — після кожної операції зберігається картинка, з картинок збирається GIF
- Бенчмарк, який порівнює всі дерева між собою і з SQLite

## Які бібліотеки використовуємо і для чого

| `matplotlib` | малювати дерева і графіки бенчмарку |
| `Pillow` (PIL) | склеювати картинки в GIF |
| `sqlite3` | для порівняння наших дерев із справжньою СУБД (вбудована в Python) |
| `argparse` | читати аргументи командного рядка (вбудована) |
| `json` | парсити дані запису, які користувач вводить (вбудована) |
| `random`, `time`, `statistics`, `gc` | для бенчмарку — генерація ключів, вимір часу, медіана (вбудовані) |

## Як запустити

### База даних

```
python main.py --tree avl
```

Замість `avl` можна підставити інше дерево:

| Дерево | Аргумент |
| :--- | :--- |
| AVL | `avl` |
| Red-Black | `rb` |
| Splay | `splay` |
| 2-3 | `2-3` |
| B-tree | `btree` |
| B+ tree | `bplus` |

Команди в програмі:

```
INSERT 101 {"name": "Mykyta"}     — додати запис
SELECT 101                        — знайти
UPDATE 101 {"name": "Anna"}       — оновити
DELETE 101                        — видалити
q                                 — вийти
```

Після виходу зберігається GIF у `results/GIFs_with_database/`.

### Бенчмарк

Найпростіше:

```
python benchmark.py --quick
```

`--quick` робить швидкий прогін на маленьких N (200, 500, 1000, 2000), щоб перевірити що все працює. Без `--quick` буде повний прогон на більших N (1000–50000), це довше.

Усі прапорці, які можна додати:

| Прапорець | Що робить |
| `--sizes 1000 5000 10000` | задати свій список розмірів N |
| `--repeats 3` | скільки разів повторити кожен прогін (береться медіана) |
| `--seed 42` | зерно для генератора випадкових ключів — щоб результати повторювались |
| `--quick` | швидка перевірка на малих N |
| `--output my_results` | в яку папку складати результати (за замовчуванням `results`) |
| `--no-avl` | не тестувати AVL-дерево |
| `--no-splay` | не тестувати Splay-дерево |
| `--no-rb` | не тестувати червоно-чорне дерево |
| `--no-23` | не тестувати 2-3 дерево |
| `--no-btree` | не тестувати B-дерево |
| `--no-bplus` | не тестувати B+ дерево |
| `--no-sqlite` | не порівнювати зі SQLite |
| `--btree-m 4` | порядок (m) для B-дерева |
| `--bplus-m 4` | порядок (m) для B+ дерева |

Прапорці можна комбінувати, наприклад:

```
python benchmark.py --sizes 1000 5000 10000 25000 --repeats 3
python benchmark.py --no-splay --no-bplus
python benchmark.py --quick --no-sqlite --output test_run
```

Результати збережуться в папці `results/` (або тій, що вкажете в `--output`).
